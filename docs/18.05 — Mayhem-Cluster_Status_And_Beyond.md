# El bug crítico encontrado

Durante las pruebas apareció un fallo muy importante:

## Síntoma

Cuando el leader entraba en estado “sleep”, el resto de nodos:

* seguían viendo al leader como vivo
* no incrementaban correctamente `last_seen`
* no disparaban failover
* podían quedarse sin leader estable

---

# Diagnóstico real

El problema NO estaba en:

* el cálculo de leader
* el TTL
* el envejecimiento de nodos
* el election algorithm

El problema real era:

## Se estaba bloqueando la ENTRADA de heartbeats

y no la salida.

Eso provocaba que:

* el nodo aislado siguiera enviando heartbeats
* algunos nodos mantuvieran timestamps frescos
* las vistas del cluster divergieran

---

# Corrección aplicada

Se introdujo:

## Estado `ISOLATED`

```python
class NodeState(str, Enum):
    BOOT = "BOOT"
    DISCOVERING = "DISCOVERING"
    STANDBY = "STAND-BY"
    ACTIVE = "ACTIVE"
    DEGRADED = "DEGRADED"
    ISOLATED = "ISOLATED"
    OFFLINE = "OFFLINE"
```
