## Deuda técnica detectada

### 1. `runtime/api_app.py` sigue demasiado cargado
Ahora mismo mezcla definición de endpoints, forwarding, control de estado del nodo y mutación de `cluster_state`, así que es el siguiente módulo evidente a refactorizar. [file:1009]

### 2. Bug claro en `runtime/dispatcher.py`
Hay una incoherencia directa en el manejo de `attempt`, porque primero se incrementa con `event.attempt = (event.attempt or 0) + 1` y justo después se resetea con `event.attempt = 0`. [file:1009]

Eso está mal, rompe la semántica de reintentos y además puede sabotear la idempotencia basada en `execution_key`. [file:1009]

### 3. Idempotencia incompleta
`runtime/worker/event_worker.py` genera `execution_key = f"{event.event_id}:{event.attempt}"`, pero el propio código anota que el chequeo “no funciona”, y eso encaja con el bug anterior de `attempt`. [file:1009]

### 4. Detalle pendiente en `runtime/context.py`
El fichero solo define `node_id = None`, pero en runtime real también se están usando `ctx.priority` y `ctx.node`, así que el módulo debería declarar explícitamente esos campos para dejar el contrato claro. [file:1009]

### 5. Import sobrante original en `runtime/node_boot.py`
En la versión del archivo adjunto aparece importado `cluster_state` en `node_boot.py`, pero no se usa. [file:1009]
